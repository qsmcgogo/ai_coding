# 工程题 2：贪吃蛇

## 题目描述

你接手了一个未完成的贪吃蛇项目。游戏引擎 `js/engine.js` 已经写好了大部分代码，
但**其中存在若干 Bug**。核心游戏逻辑 `js/logic.js` 中只有空的函数声明，
需要你来实现。

你的任务：
1. **阅读 `engine.js`**，理解它是如何调用 `logic.js` 中各函数的
2. **实现 `logic.js`** 中所有函数（函数的作用需要你从 `engine.js` 的调用上下文推断）
3. **修复 `engine.js`** 中的 Bug（共有 3 处逻辑错误，不是语法错误）

## 项目结构

| 文件 | 说明 |
|------|------|
| `index.html` | 游戏页面 [锁定] |
| `css/style.css` | 样式 [锁定] |
| `js/engine.js` | 游戏引擎 [**可编辑** - 需修复 Bug] |
| `js/logic.js` | 核心逻辑 [**可编辑** - 需实现] |
| `tests/test.js` | 自动化测试 [锁定] |

## 提示

- `engine.js` 中的 Bug 都是**逻辑层面**的错误，代码能运行但行为不正确
- `logic.js` 的函数签名（函数名和参数名）已给出，你需要从 `engine.js` 的
  调用代码推断每个函数应该做什么、返回什么
- 运行测试可以帮助你验证实现是否正确
- 建议先通读 `engine.js` 全部代码再动手

## 操作说明

实现完成后，可通过 **预览运行** 按钮试玩游戏：

- 方向键 ←→↑↓：控制蛇的移动方向
- P 键：暂停/继续
- 任意键：开始新游戏

## 测试

点击平台上方的 **运行测试** 按钮即可执行自动化测试。
测试不仅验证 `logic.js` 的函数功能，也会验证游戏整体逻辑的正确性。
